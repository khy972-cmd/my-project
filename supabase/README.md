# Supabase 구조/운영 가이드

이 폴더는 Supabase(데이터베이스/정책/RPC/스토리지) 관련 산출물을 관리합니다.

## 핵심 원칙(중요)

- **기존 `supabase/migrations/*.sql` 파일은 이동/이름변경 금지**
  - Supabase는 마이그레이션을 **파일명(시간순)** 으로 실행합니다.
  - 과거 파일을 이동/리네임하면 적용 순서가 바뀌거나, 누락/중복 적용으로 운영 장애가 발생할 수 있습니다.
- 변경은 **추가(additive)** 중심으로 합니다.
  - 새 테이블/함수/정책은 `CREATE ... IF NOT EXISTS` 또는 존재 확인 후 생성 패턴을 권장합니다.
- UI는 **스키마 미적용 상태에서도** 깨지지 않도록(빈 상태/가이드 문구) 설계합니다.

## 디렉터리 설명

- `migrations/`
  - 실제 배포/적용되는 SQL 마이그레이션 파일.
  - **반드시 평평한 구조 유지**를 권장합니다(폴더 분리 X).
- `sql/` (선택)
  - 재사용 가능한 SQL 조각/템플릿/초안 보관용.
  - 여기의 파일은 자동 적용되지 않습니다.
  - 실제 반영은 `migrations/`에 **새 마이그레이션 파일**로 추가하세요.

## 파일명 규칙(권장)

새 마이그레이션은 아래 규칙을 권장합니다.

- `YYYYMMDDHHMMSS_<module>_<action>.sql`
  - 예) `20260310160000_admin_system_configs.sql`

## 적용(예시)

프로젝트 환경에 따라 다르지만, 일반적으로는 Supabase CLI를 사용합니다.

1) 로그인

```bash
supabase login
```

2) 프로젝트 연결(필요 시)

```bash
supabase link --project-ref <project-ref>
```

3) 마이그레이션 적용

```bash
supabase db push
```

> 주의: 위 명령은 로컬/원격 환경 설정에 따라 동작이 달라질 수 있습니다.
